# Браузерный расчётчик электрики v2.0: один HTML-файл с подбором кабеля, автоматов, УЗО и сметой — без бэкенда

*Теги: javascript, электротехника, веб-разработка, open source, инструменты*

---

## Предыстория

Когда делаешь ремонт или помогаешь кому-то с электрикой, неизбежно возникает вопрос: какой кабель брать, какой автомат ставить, сколько всего нужно на щиток?

Гуглишь — находишь либо огромные Excel-таблицы непонятного происхождения, либо калькуляторы с одним полем «введите мощность», которые выдают одно число. Ни то, ни другое не помогает составить реальную картину.

Я решил сделать инструмент, который даёт полный ответ: группы нагрузки, кабели, автоматы, УЗО, схему щитка и смету — всё сразу. И чтобы работало без регистрации, без установки, без интернета.

Результат: **ElectroKit** — один HTML-файл, никаких зависимостей.

**[Попробовать онлайн](https://darkenamber.github.io/electrokit/electrokit.html)** | **[GitHub](https://github.com/darkenamber/electrokit)**

## Что изменилось в v2.0

После публикации первой версии читатели нашли несколько реальных ошибок в расчётной логике. Это главное что исправлено в v2.

### Логика подбора кабеля была неверной

В v1 автомат выбирался по расчётному току, кабель подбирался под автомат. Это нарушение ПУЭ — автомат должен защищать кабель, а не наоборот. Конкретный пример: розеточная группа 4500 Вт давала ток 19.6А, попадала в ветку `a <= 25` и получала **2.5мм² + C25**. Но допустимый ток 2.5мм² — 21А. C25 этот кабель не защищает.

В v2 логика исправлена: автомат по рабочему току, кабель по допустимому току с запасом.

### Схема мокрых зон — логическое противоречие

В v1 на мокрую зону одновременно ставилось УЗО зоны и дифавтоматы на линии за ним. Это противоречие: если на линии уже стоит дифавтомат, УЗО зоны избыточно. В v2 две корректные схемы на выбор в настройках.

### Духовка получала 2.5мм²

Минимально допустимое сечение для духового шкафа и варочной панели — 4мм² независимо от расчётного тока. В v1 этого ограничения не было.

### Противопожарное УЗО стояло по умолчанию

Без реле напряжения и УЗИП противопожарное УЗО 100мА не даёт реальной защиты. В v2 убрано из дефолтной схемы, доступно как опция «Расширенная защита».

## Что умеет

Пользователь добавляет комнаты, в каждой — приборы с типом и количеством. Нажимает «Рассчитать». Инструмент делает следующее:

**Разбивает на группы.** Освещение отдельно, розетки отдельно, каждый мощный прибор на свою линию. Кондиционеры — по одной группе на каждый блок.

**Подбирает кабель.** ВВГнг по допустимому току (ПУЭ табл. 1.3.4). Автомат выбирается по рабочему току, кабель — по допустимому току с запасом ×1.25 для выделенных линий. Минимальные сечения: розетки 2.5мм², духовка и варочная 4мм², комбо плита+духовка 6мм².

**Подбирает автомат.** По рабочему току нагрузки. Класс B для освещения, C для остального. Автомат никогда не превышает допустимый ток кабеля. Цветовая индикация: зелёный ≤16А, жёлтый ≤32А, красный >32А.

**Определяет мокрые зоны.** Автоматически по ключевым словам в названии комнаты. Две схемы защиты на выбор: схема А — дифавтомат на каждую линию (по умолчанию), схема Б — УЗО на зону с расчётным номиналом по сумме токов плюс обычные автоматы. Ванная/душ/баня — 10мА, кухня/санузел — 30мА.

**Показывает допустимый ток кабеля.** Под сечением в таблице — «доп. 21А» с тултипом про условия прокладки. Пользователь видит на чём основан выбор.

**Показывает коэффициент спроса явно.** В итогах: `18 240 × Kd 0.7 (квартира) = 12 768 Вт`. Объясняет почему 100А вводной не нужен.

**Считает падение напряжения.** Если ввести длину линии, сразу появляется ΔU% — зелёный до 4%, жёлтый до 5%, красный с рекомендацией увеличить сечение.

**Проверяет селективность.** Если групповой автомат равен или больше вводного — красная ошибка с объяснением. Если разница меньше одной ступени — жёлтое предупреждение.

**Генерирует схему щитка.** Визуальная DIN-рейка в соответствии с выбранной схемой защиты. Количество модулей считается с учётом схемы А или Б и наличия противопожарного УЗО.

**Составляет смету.** Кабель по сечениям, автоматы по номиналам, дифавтоматы или УЗО в зависимости от схемы, розетки, выключатели, подрозетники. Работы: прокладка, штробление, установка, сборка щитка. Итого в вашей валюте.

**Даёт рекомендации.** Если вводной загружен >80%, если на розеточной группе слишком много точек, если щиток требует >16 модулей.

## Технические решения

### Почему один файл

Это сознательное архитектурное решение, а не лень разбить на модули.

Один файл — это:
- Скинуть коллеге в Telegram одним сообщением
- Открыть на любом устройстве без установки
- Работать на объекте без интернета
- Нет проблем с обновлениями CDN

Единственная «зависимость» — Google Fonts для шрифтов. При отсутствии интернета браузер просто использует системный шрифт.

### Хранение данных

Всё в `localStorage`. Два ключа:

```javascript
localStorage.setItem('ek_proj', JSON.stringify(project)); // проект
localStorage.setItem('ek_cfg',  JSON.stringify(settings)); // цены и настройки
```

Проект сохраняется при каждом изменении. Настройки — при каждом изменении цены. Никаких кнопок «Сохранить».

Дополнительно — импорт/экспорт в JSON. Можно передать проект коллеге или хранить несколько объектов как отдельные файлы.

### Логика подбора кабеля (v2)

Таблица допустимых токов по ПУЭ табл. 1.3.4 (медь, одиночная прокладка, +25°C):

```javascript
const CABLE_TABLE = [
  { sec: '1.5', amp: 15, maxBrk: 10 },
  { sec: '2.5', amp: 21, maxBrk: 16 },
  { sec: '4.0', amp: 27, maxBrk: 25 },
  { sec: '6.0', amp: 34, maxBrk: 32 },
  { sec: '10',  amp: 50, maxBrk: 50 }
];

function cableForLoad(watts, lineType, voltage, minSectionOverride) {
  const cableMargin = (lineType === 'dedicated' || lineType === 'ac') ? 1.25 : 1.0;
  const breakerI = watts / voltage;                 // ток для автомата — рабочий
  const cableI   = (watts * cableMargin) / voltage; // ток для кабеля — с запасом

  // 1. Автомат по рабочему току
  let breaker = STD_BREAKERS.find(b => b >= breakerI);

  // 2. Кабель — наименьшее сечение, которое выдерживает ток с запасом
  //    И допускает выбранный автомат (maxBrk >= breaker)
  let rowIdx = CABLE_TABLE.findIndex(r =>
    parseFloat(r.sec) >= minSec && r.amp >= cableI && r.maxBrk >= breaker
  );

  return { section: row.sec, breaker, allowedA: row.amp };
}
```

Пример который был сломан в v1: розетки 4500 Вт → рабочий ток 19.6А → автомат C25 → кабель нужен с допустимым током ≥ 19.6А и maxBrk ≥ 25 → **4.0мм² (доп. 27А, maxBrk 25)**. В v1 это давало 2.5мм² + C25, что неверно.

### Схемы защиты мокрых зон (v2)

```javascript
// Схема А (по умолчанию): дифавтомат на каждую линию
// Схема Б: одно УЗО на зону + обычные автоматы

function buildWetZones(wetGroups) {
  // Группировка по комнате + уставке мА
  // Ванная (10мА) и кухня (30мА) — всегда разные зоны
  const map = new Map();
  for (const g of wetGroups) {
    const key = g.room + '|' + g.uzomA;
    if (!map.has(key)) map.set(key, { room: g.room, uzomA: g.uzomA, groups: [] });
    map.get(key).groups.push(g);
  }
  // Номинал УЗО = ближайший стандартный >= суммы токов линий за ним
  for (const z of map.values()) {
    z.sumA = z.groups.reduce((s, g) => s + g.watts / voltage, 0);
    z.uzoRating = UZO_STD.find(r => r >= z.sumA) || 100;
  }
  return [...map.values()];
}
```

### Расчёт падения напряжения

```javascript
function voltDrop(watts, lengthM, sectionStr) {
  // ΔU% = (2 × L × I × ρ) / (S × U) × 100
  // ρ = 0.0175 Ω·мм²/м для меди
  const I = watts / voltage;
  return (2 * lengthM * I * 0.0175) / (parseFloat(sectionStr) * voltage) * 100;
}
```

Если ΔU > 5% — красный индикатор и подсказка с увеличенным сечением.

### Автоматическое определение мокрых зон

```javascript
const WET_KW     = ['ван', 'санузел', 'туалет', 'кухн', 'баня', 'душ',
                    'bathroom', 'kitchen', 'toilet', 'wc'];
const EXTRA_WET_KW = ['ван', 'баня', 'душ', 'bathroom', 'shower'];

function isWetRoom(name)      { return WET_KW.some(k => name.toLowerCase().includes(k)); }
function isExtraWetRoom(name) { return EXTRA_WET_KW.some(k => name.toLowerCase().includes(k)); }
```

Ванная/баня/душ → 10мА. Кухня/санузел → 30мА. Никакого ручного флага.

## Что не считает

Честно о границах инструмента:

- Трёхфазная сеть (планируется в v3)
- Реактивная мощность и cos φ
- Термическая стойкость кабеля при КЗ
- Уставки по времени срабатывания автоматов
- Расчёт токов КЗ
- Селективность по токам КЗ между модульными автоматами — её нет, это физика

ElectroKit — инструмент для **предварительной оценки**, не замена проектной документации. Нормативная база: ПУЭ табл. 1.3.4, СП 256.1325800.2016, ГОСТ Р 50571.5.52-2011.
